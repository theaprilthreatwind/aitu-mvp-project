# MDEatOS 

RESTful API сервис для системы заказа еды из ресторанов.
Реализована ролевая модель (Администратор, Клиент), управление меню, оформление заказов и получение статистики.

## Технологический стек
* **Язык:** Java 23+
* **Фреймворк:** Spring Boot (Web)
* **База данных:** In Memory (ConcurrentHashMap / ArrayList)
* **Аутентификация:** JWT Токены
* **Инструменты:** Maven, Postman, ngrok

## Функционал
* **Аутентификация:** Регистрация и вход (выдача JWT токена).
* **Публичный доступ:** Просмотр списка ресторанов, категорий и меню.

* **Роль "MANAGER":**
    * CRD операции для категорий и блюд
    * Просмотр всех заказов
    * Получение статистики (доход (общий и дневной), количество клиентов)
* **Роль "CLIENT":**
    * Оформление заказа.
    * Просмотр истории своих заказов.

## Установка и запуск (через терминал)

Кстати, проект не требует настройки базы данных. Просто скачайте и запустите.
Вся сборка проекта проходила в IntelliJ IDEA Community Edition.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/theaprilthreatwind/aitu-mvp-project.git

    ```
2.  **Запустите приложение:**
    ```bash
    ./mvnw spring-boot:run
    ```
3.  **Подключите доступ к фронтенду:**
    ```bash
    ngrok config add-authtoken 38DU12yBHRx4Jo34wRMlPxpFZP8_89xoDdkLsa2iec1G7PE2q
    ```
    ```bash
    ngrok http 8080
    ```
4.  **Скопируйте Forwarding URL; например, https://obliging-greatly-monkfish.ngrok-free.app/**
   
**Важно:** Так как данные хранятся в памяти проекта, при перезапуске приложения все созданные пользователи, блюда и заказы будут сброшены.

##  API эндпоинты

### Аутентификация (Вход/Регистрация)
| Метод | URL | Описание |
|---|---|---|
| `POST` | `/register` | Регистрация (Role: "CLIENT" или "MANAGER") |
| `POST` | `/login` | Вход в систему (возвращает токен) |

### Меню и Рестораны (Общий доступ)
| Метод | URL | Описание |
|---|---|---|
| `GET` | `/api/categories` | Список всех категорий |
| `GET` | `/api/restaurants` | Список ресторанов |
| `GET` | `/api/categories/{id}/dishes` | Блюда конкретной категории |
| `GET` | `/api/menu/{name}` | Меню конкретного ресторана |

### Администратор (Требуется токен)
| Метод | URL | Описание |
|---|---|---|
| `POST` | `/api/admin/category` | Создать категорию |
| `POST` | `/api/admin/dish` | Создать блюдо (с привязкой к categoryId) |
| `GET` | `/api/admin/orders` | Список всех заказов |
| `DELETE`| `/api/admin/dish/{id}` | Удалить блюдо |
| `DELETE`| `/api/admin/category/{id}` | Удалить категорию (каскадно) |
| `GET` | `/api/admin/stats` | Статистика (доход, клиенты) |

###  Клиент (Требуется токен)
| Метод | URL | Описание |
|---|---|---|
| `POST` | `/api/client/order` | Оформить заказ (список ID блюд) |
| `GET` | `/api/client/my-orders` | История заказов |
